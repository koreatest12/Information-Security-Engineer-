# 정보보안기사 과목별 상세 학습 가이드 (필기/실기)

> 적용 범위: **필기 5과목 + 실기 1과목**
> 
> 필기 과목: 시스템보안, 네트워크보안, 어플리케이션보안, 정보보안일반, 정보보안관리 및 법규
> 
> 실기 과목: 정보보안 실무

---

## 0) 시험 구조 핵심 요약

- **필기**: 객관식 4지선다, 총 2시간 30분, 과목당 20문항
- **필기 합격**: 과목당 40점 이상 + 전과목 평균 60점 이상
- **실기**: 필답형, 3시간
- **실기 합격**: 100점 만점 중 60점 이상

---

## 1) 시스템보안

### 1-1. 핵심 범위
- 운영체제 보안 구조(커널/유저모드, 프로세스/메모리 보호)
- 식별/인증/인가, 계정 및 권한 관리
- 접근통제 정책/모델(DAC, MAC, RBAC, BLP, Biba 등)
- Linux/Unix/Windows 보안 설정
- 시스템 로그/감사/모니터링
- 시스템 공격 기법(권한상승, 버퍼오버플로우, 루트킷 등)
- 하드닝/패치/무결성 점검

### 1-2. 시험에 자주 나오는 디테일
- Linux 주요 파일: `/etc/passwd`, `/etc/shadow`, `/etc/group`
- 특수 권한: SUID/SGID/Sticky bit
- 인증 모듈: PAM
- 계정/로그 명령: `who`, `w`, `last`, `pwck`, `lsof`, `netstat`(또는 `ss`)
- Windows: SID, SAM, NTLM, Kerberos, 이벤트 로그

### 1-3. 필기-실기 연계 포인트
- “취약 설정 식별 → 보완안 제시” 형태 연습
- 로그 기반 침해 징후 해석(비정상 로그인, 권한변경, 서비스 이상)

### 1-4. 우선순위 학습 순서
1. 계정/권한/인증
2. 접근통제 정책·모델
3. OS별 보안설정
4. 로그/감사
5. 공격기법과 대응

---

## 2) 네트워크보안

### 2-1. 핵심 범위
- OSI 7계층, TCP/IP, 주요 프로토콜(TCP/UDP/IP/ICMP/ARP/DNS)
- 네트워크 공격(스캐닝, 스푸핑, 스니핑, 세션 하이재킹)
- DoS/DDoS 유형(SYN Flood, UDP Flood 등)
- 보안 프로토콜(SSL/TLS, IPSec, SSH, VPN)
- 네트워크 보안 장비(FW, IDS, IPS, WAF, NAC, UTM)
- 패킷/로그 분석 기본

### 2-2. 시험에 자주 나오는 디테일
- TCP 3-way handshake 흐름
- IDS vs IPS 차이(탐지/차단)
- WAF의 적용 계층과 목적
- VPN 터널링의 개념과 목적
- ARP Spoofing, DNS Spoofing의 원리와 대응

### 2-3. 필기-실기 연계 포인트
- 공격 유형과 방어 수단의 매핑(공격-탐지-차단)
- 패킷/로그에서 공격 흔적 판단 근거 제시

### 2-4. 우선순위 학습 순서
1. TCP/IP + 프로토콜 역할
2. 공격 유형별 원리
3. 보안 장비 비교
4. 보안 프로토콜
5. 로그/패킷 해석

---

## 3) 어플리케이션보안

### 3-1. 핵심 범위
- HTTP/HTTPS, 쿠키/세션, 인증/인가
- 웹 취약점(SQL Injection, XSS, CSRF)
- 파일 업로드/경로 조작/인증 우회/세션 관리 취약점
- DB 보안, DNS/메일/FTP 보안
- 시큐어코딩(입력값 검증, 예외처리, 에러노출 통제)
- 개발보안 프로세스(요구사항~구현~점검)

### 3-2. 시험에 자주 나오는 디테일
- SQL Injection 대응: 파라미터 바인딩, 입력값 검증
- XSS 대응: 출력 인코딩, 입력 검증, 보안 헤더
- CSRF 대응: 토큰, SameSite Cookie 설정
- Broken Access Control류 문제(서버 측 권한검증)

### 3-3. 필기-실기 연계 포인트
- “취약점 이름”보다 “원인-영향-대응” 구조로 답안 작성
- 소스/설정 예시를 보고 취약 여부 판단하는 연습

### 3-4. 우선순위 학습 순서
1. 웹 구조(요청/응답, 세션)
2. 3대 웹 취약점(SQLi/XSS/CSRF)
3. 인증/인가/세션 취약점
4. 시큐어코딩 원칙
5. 서비스별 보안(DB/메일/DNS/FTP)

---

## 4) 정보보안일반

### 4-1. 핵심 범위
- 정보보호 목표(CIA)와 보안서비스(인증, 부인방지 등)
- 인증과 접근통제 기본 이론
- 암호학 기초: 대칭키/공개키/키교환
- 전자서명, 해시, MAC/HMAC
- PKI 구조(CA, RA, 인증서, CRL, OCSP)

### 4-2. 시험에 자주 나오는 디테일
- 대칭키 vs 공개키 장단점
- 암호화 vs 전자서명 목적 차이
- 해시 vs MAC 차이
- RSA/ECC/Diffie-Hellman 역할 구분
- 인증서 폐기/상태검증(CRL/OCSP)

### 4-3. 필기-실기 연계 포인트
- “기밀성/무결성/인증/부인방지” 제공 속성 매칭
- 암호·서명·해시를 절차도 형태로 설명하는 연습

### 4-4. 우선순위 학습 순서
1. CIA + 보안서비스
2. 대칭키/공개키
3. 전자서명/PKI
4. 해시/MAC
5. 인증·접근통제 이론 정리

---

## 5) 정보보안관리 및 법규

### 5-1. 핵심 범위
- 정보보호 거버넌스, 정책, 조직/역할
- 자산 식별, 위험평가, 통제 선정
- 보호대책 운영/모니터링/감사/개선
- 인증제도(ISMS-P 등) 이해
- 개인정보보호 및 정보보호 관련 법규

### 5-2. 시험에 자주 나오는 디테일
- 위험 처리 전략(회피/감소/전가/수용)
- 관리적/물리적/기술적 보호조치 구분
- ISMS-P의 목적·절차·구성
- 법규의 목적조항, 의무주체, 금지행위 구분

### 5-3. 필기-실기 연계 포인트
- 절차형 서술: 정책→위험평가→통제→점검→개선
- 법규형 서술: “대상/의무/위반 시 리스크” 프레임

### 5-4. 우선순위 학습 순서
1. 위험관리 프로세스
2. ISMS-P 체계
3. 개인정보 보호 핵심 원칙
4. 전자서명/전자금융/정보통신망 관련 핵심 조문 취지
5. 윤리·컴플라이언스

---

## 6) 실기(정보보안 실무) 연결 학습법

- 실기는 **암기만으로 합격이 어려움**
- 필기 개념을 “실무 답안 문장”으로 전환해야 함

### 실기 답안 기본 템플릿
1. 취약점/위협 정의
2. 발생 원인
3. 영향 범위
4. 탐지/확인 방법
5. 대응 및 재발방지

### 실기에서 점수 잘 나오는 습관
- 키워드 중심 단문 작성(장황한 서술 지양)
- 비교 문제는 표 형태로 정리 후 암기
- 로그/패킷/구성도 문제는 근거 문장 포함

---

## 7) 8주 학습 로드맵(압축)

- **1~2주차**: 시스템보안 + 정보보안일반(암호)
- **3~4주차**: 네트워크보안 + 어플리케이션보안
- **5주차**: 관리 및 법규 집중
- **6주차**: 전과목 기출 1회전(오답노트)
- **7주차**: 취약 단원 보완 + 기출 2회전
- **8주차**: 실기형 서술 연습 + 최종 점검

---

## 8) 마지막 체크리스트

- 과목명은 반드시 공식 명칭으로 기억
- 예전 명칭(암호학/법규 단독 과목 등)과 혼동 금지
- 법규는 시험 직전 최신 시행일 확인
- 기출은 “정답 암기”보다 “오답 이유” 중심 복습
- 실기 대비를 필기 단계부터 병행
